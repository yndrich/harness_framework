# 출력/UX 가이드 (CLI + 엑셀)

> 이 프로젝트는 웹 UI 가 없다. 'UI' = 명령행 사용성과 엑셀 산출물의 가독성이다.
> (원래 템플릿의 웹/Tailwind 디자인 규약은 이 프로젝트에 해당 없음.)

## 디자인 원칙
1. 도구처럼 보여야 한다 — 분석가가 매일 여는 작업용 시트지 발표 자료가 아니다.
2. 모든 수치는 출처로 되돌아갈 수 있어야 한다 (Provenance). 출처 없는 값을 보여주지 않는다.
3. 자동화가 애매한 곳을 숨기지 않는다 — 색으로 드러내고 Review 시트에 모은다.

## CLI 규약
- 기업마다 한 줄 진행 요약: `티커: 회사명 | years 2021–2025 | review flags: N`.
- 실패는 그 기업만 건너뛰고(`[skip]`/`[error]`) 나머지는 계속 — 한 기업 때문에 전체가 죽지 않는다.
- 부수효과는 명시적으로: 마지막에 출력 경로와 생성된 시트 구성을 알린다.
- 인자: `tickers...`, `--years`, `--output`, `--user-agent`, `--refresh`, `--no-cache`, `--cache-dir`.

## 엑셀 레이아웃 규약
- 제표 시트: 행 = 표준 항목(고정 순서), 열 = 연도(최근이 오른쪽). 다기업이면 기업 블록을 가로로.
- 헤더 3행 고정(틀 고정), 1열(항목명) 고정. 숫자는 천단위 구분(`#,##0`), EPS 는 `#,##0.00`.
- 워크북 시트 구성(고정): `Income Statement` · `Balance Sheet` · `Cash Flow` ·
  `Comparison`(핵심지표 + FCF·마진) · `Review`(검토 목록) · `Provenance`(출처 추적).

## 검토 색상 (의미색 — 이 3개만 사용)
| 플래그 | 의미 | 색(RGB) |
|--------|------|--------|
| AMBIGUOUS | 후보 태그 충돌 | `FFF2CC` (연노랑) |
| RESTATED | 재작성됨 | `FCE4D6` (연주황) |
| GAP | 중간연도 누락 | `F8CBAD` (연빨강) |
| (헤더) | 열/연도 헤더 | `D9E1F2` (옅은 파랑) |
| (기업명) | 기업 블록 머리 | `BDD7EE` (파랑) |

## 안티패턴 — 하지 마라
| 금지 | 이유 |
|------|------|
| 애매한 값을 플래그 없이 채택 | 사람 검토 가능성이 핵심 요구 — 조용한 추정은 신뢰를 깬다 |
| 출처(태그/accession) 없는 값 출력 | 추적 불가능한 수치는 검증할 수 없다 |
| 한 셀에 단위 뒤섞기(달러/주식/주당) | 비교가 깨진다. 항목별 단위·서식을 고정 |
| 의미 없는 장식 색·테두리 남발 | 색은 검토 플래그에만. 나머지는 무채색 |
| 차트/조건부서식 등 xlsx.py 미지원 기능 가정 | 작성기는 값·스타일·병합·틀고정만 지원 |
